## 1. Цель

**GoAcademy** — веб-платформа для системного изучения языка программирования **Go**: от новичка до уверенного разработчика. Платформа объединяет в одном месте контент разных форматов, отслеживает личный прогресс ученика и мотивирует его геймификацией.

## 2. Целевая аудитория

- Начинающие и средние разработчики, изучающие Go.
- Мультиязычная аудитория: интерфейс на **русском, английском, узбекском, японском**.

## 3. Ключевые ценности продукта

- Контент разных форматов в одном месте, разнесённый по понятным разделам.
- Прозрачный личный прогресс: что просмотрено, что решено, динамика по дням.
- Мотивация через очки (XP), уровни, бейджи, серии (streaks) и рейтинг.
- Возможность учиться структурированно по трекам или свободно по отдельным материалам.

## 5. Авторизация и пользователи

- Регистрация/вход по **email + паролю**.
- Верификация email и сброс пароля по токенам.
- Пароли — только хэш (argon2id).
- JWT **access** (короткоживущий) + **refresh** (httpOnly cookie, ротация).
- Роли: `student` (по умолчанию) и `admin`.
- Сторонний OAuth — не на старте (закладывается на будущее).
- Email-отправка — через интерфейс с заглушкой (логирует токен), реальный SMTP позже.

## 6. Технологический стек

**Backend (Go):** Go (последняя стабильная), `chi`, `pgx` + `sqlc`, `golang-migrate`, `log/slog`, JWT, конфиг из ENV с валидацией, `testing` + `testify`.

**База данных:** PostgreSQL (последняя стабильная мажорная), нормализованная схема, индексы, внешние ключи, транзакции для XP/streak.

**Frontend:** React + TypeScript + Vite, TanStack Query, TanStack Router/React Router, Tailwind CSS + shadcn/ui, i18next (RU/EN/UZ/JA), Markdown с подсветкой Go.

**Инфраструктура:** Docker Compose (`backend`, `frontend`, `postgres`, позже `code-runner`). Запуск через `docker compose up`.

> Стек — базовая рекомендация. Изменения компонентов фиксируются в `decisions` (STATUS.yaml) и в [ARCHITECTURE.md](./ARCHITECTURE.md).

## 7. Нефункциональные требования

- **Безопасность:** argon2id для паролей; хранение хэшей токенов; параметризованные запросы (sqlc); санитизация Markdown (XSS); CSRF для cookie-сессий; rate limiting на auth; изоляция исполнения кода.
- **Качество:** код компилируется и проходит линтер; тесты на бизнес-логику (XP, streak, проверка квизов, прогресс трека); единый формат ошибок API.
- **Наблюдаемость:** структурированные логи через `slog` без утечки секретов.
- **i18n:** все строки UI через ключи i18next; локаль хранится в профиле; стратегия мультиязычного контента — см. ARCHITECTURE.md (решение D-008).
- **Доступность:** адаптивная вёрстка (desktop + mobile), a11y.
- **Деплой:** воспроизводимый запуск через Docker Compose; документированные переменные окружения; задокументированный порядок миграций.

## 8. Границы MVP vs поздних фаз

- **MVP включает:** весь контент кроме автосудьи и реального исполнения кода; ручная отметка решения задач; геймификация; кабинет; админка; frontend всех разделов; i18n.
- **Поздние фазы:** изолированный раннер + песочница (CHAPTER 16), онлайн-судья (CHAPTER 17). Интерфейсы под них закладываются заранее, чтобы встроились без переделок.
